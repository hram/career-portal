# Шаг 4 — Опыт работы и проекты (ключевой редактор)

## Контекст
Шаги 1-3 выполнены. Профиль работает.  
Это самый важный экран портала — редактор опыта работы с вложенными проектами.

---

## Задача
Создать полноценный редактор: список компаний → раскрывающиеся проекты внутри каждой.

---

## Router: `app/routers/jobs.py`

```
GET    /jobs                        → список всех мест работы
POST   /jobs                        → создать новое место работы
GET    /jobs/{job_id}               → детальная страница одного места работы
POST   /jobs/{job_id}               → обновить место работы
DELETE /jobs/{job_id}               → удалить (+ каскадно все проекты)

POST   /jobs/{job_id}/projects           → создать проект
POST   /jobs/{job_id}/projects/{pid}     → обновить проект
DELETE /jobs/{job_id}/projects/{pid}     → удалить проект

POST   /jobs/reorder                → принять новый порядок jobs (JSON: [{id, sort_order}])
POST   /jobs/{job_id}/projects/reorder  → аналогично для проектов
```

---

## Template: `app/templates/jobs.html` — список компаний

Каждое место работы — карточка с:
- **Компания** (крупно) + должность
- Период: "март 2021 — present"
- Количество проектов: `5 проектов`
- Кнопки: `Открыть`, `Удалить`
- Drag handle для ручной сортировки (иконка ⠿)

Вверху: кнопка `+ Добавить место работы` → inline форма или модалка.

**Форма добавления:**
```
Компания *
Должность *
Город (необязательно)
Период с: [____]  по: [____]  [x] по настоящее время
Заметки (необязательно) — большой textarea
```

---

## Template: `app/templates/job_detail.html` — одно место работы

### Верхняя часть — данные о работе
Инлайн-редактируемые поля (click to edit):
- Компания, должность, период, город
- `raw_notes` — большой textarea: `Общие заметки про это место работы. Команда, стек, атмосфера, почему ушёл — пиши как угодно`

### Нижняя часть — проекты (главное)

Каждый проект — раскрывающаяся карточка:

**Свёрнутый вид:**
```
[⠿] [▶] Название проекта   Python, FastAPI, Redis   тех лид   [ Изменить ] [ ✕ ]
```

**Развёрнутый вид — форма редактирования:**
```
Название проекта *
[_________________________________]

Описание (сырые заметки — пиши как угодно) *
[большой textarea — минимум 6 строк]
[Здесь можно писать как угодно: что делал, зачем, что сломалось,
 как починили, что узнал. ИИ сам разберётся.]

Стек / технологии
[_________________________________]
[подсказка: Python, FastAPI, PostgreSQL, Redis, Docker — через запятую или как угодно]

Результаты и метрики (необязательно)
[textarea]
[подсказка: "ускорили в 2 раза", "запустили на 50к пользователей", "сократили время деплоя с часа до 5 минут"]

Моя роль          Размер команды
[__________]      [__________]

[ Сохранить ]  [ Отмена ]
```

**Кнопка "+ Добавить проект"** — появляется снизу списка проектов.

## UX детали

### Автосохранение
- `raw_description`, `raw_notes` — автосохранение через 1.5 сек debounce
- Индикатор состояния: `● Сохраняется...` → `✓ Сохранено`

### Пустое состояние проектов
Если проектов нет:
```
Нет проектов для этой компании.
Добавь хотя бы один — или загрузи резюме, ИИ заполнит автоматически.
[ + Добавить проект ]
```

### Drag & drop сортировка
Используй нативный HTML5 drag & drop (без библиотек).  
После drop — POST /jobs/reorder или /projects/reorder с новым порядком.
