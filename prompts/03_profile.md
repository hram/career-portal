# Шаг 3 — Страница профиля

## Контекст
Шаги 1-2 выполнены. БД работает, модели созданы.  
Теперь делаем страницу редактирования личных данных профиля.

---

## Задача
CRUD страница профиля — личные данные и свободное описание себя.

---

## Router: `app/routers/profile.py`

```
GET  /profile          → показать форму профиля (заполненную текущими данными)
POST /profile          → сохранить изменения → redirect на /profile с flash "Сохранено"
```

Всегда работаем с единственным профилем (id=1). Никакой авторизации.

---

## Template: `app/templates/profile.html`

Наследует `base.html`. Страница разделена на две колонки:

### Левая колонка — структурированные данные
Форма с полями:
- **Имя** (full_name) — крупно, это заголовок резюме
- Email, Телефон, Город
- LinkedIn URL, GitHub URL, Telegram
- Кнопка `Сохранить`

### Правая колонка — "О себе" (большое поле)
- Большой textarea для `summary_raw`
- Placeholder: `Пиши как угодно. Кто ты, чего хочешь, что тебя отличает. Это не для резюме — это сырые данные для ИИ. Например: "10 лет в Android, последние 3 года тимлид, устал от корпораций, хочу в продуктовую команду до 50 человек, не готов к полному remote..."`
- Hint под полем: `Это поле только для тебя. ИИ-агент будет использовать эти данные при генерации резюме.`
- Автосохранение через 2 секунды после остановки ввода (debounce)

---

## UX детали
- После сохранения flash: `✓ Профиль обновлён`
- Если email или телефон пустые — мягкое предупреждение (не блокирует сохранение): `Добавь контакты — они попадут в резюме`
- Заголовок страницы: имя профиля если заполнено, иначе `Мой профиль`
