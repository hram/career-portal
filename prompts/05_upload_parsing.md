# Шаг 5 — Загрузка и парсинг существующего резюме

## Контекст
Шаги 1-4 выполнены. Редактор работает.  
Теперь самое важное для старта — загрузить существующее резюме и не вводить всё с нуля.

---

## Задача
Upload PDF или DOCX → извлечь текст → Claude API парсит структуру → заполняет БД.  
Пользователь видит что получилось и может поправить вручную.

---

## Router: `app/routers/upload.py`

```
GET  /upload          → страница загрузки
POST /upload          → принять файл, запустить парсинг, redirect на /upload/preview/{job_id}
GET  /upload/preview  → показать что распарсилось, предложить сохранить или отменить
POST /upload/confirm  → сохранить распарсенные данные в БД
```

**Промпт для Claude API (внутри parser.py):**
```
Ты парсер резюме. Получи текст резюме и верни ТОЛЬКО валидный JSON без markdown.

Структура JSON:
{
  "profile": {
    "full_name": "",
    "email": "",
    "phone": "",
    "location": "",
    "linkedin_url": "",
    "github_url": "",
    "telegram": "",
    "summary_raw": ""
  },
  "jobs": [
    {
      "company": "",
      "position": "",
      "location": "",
      "start_date": "",
      "end_date": "",
      "is_current": false,
      "raw_notes": "",
      "projects": [
        {
          "name": "",
          "raw_description": "",
          "tech_stack": "",
          "results_raw": "",
          "my_role": "",
          "team_size": ""
        }
      ]
    }
  ],
  "skills_raw": "",
  "education": [
    {
      "institution": "",
      "degree": "",
      "field": "",
      "start_year": "",
      "end_year": "",
      "raw_notes": ""
    }
  ]
}

Правила:
- Если в резюме есть список проектов под компанией — каждый проект в projects[]
- Если описание обязанностей единым блоком — помести в raw_notes компании, projects оставь []
- Сохраняй оригинальный текст, не переформулируй
- start_date и end_date — строки как в резюме ("март 2021", "2021", "01.2021")
- is_current: true если "по настоящее время", "present", "н.в." и т.д.
- Если поле не найдено — пустая строка, не null

Текст резюме:
{resume_text}
```

## Template: `app/templates/upload.html`

### Страница загрузки
- Drag & drop зона для файла (PDF или DOCX)
- Или кнопка `Выбрать файл`
- Максимальный размер: 10 МБ
- После выбора показывать имя файла
- Кнопка `Загрузить и распарсить`
- Прогресс-индикатор пока идёт парсинг: `🤖 Читаю резюме...`

**Важное предупреждение на странице:**
```
⚠️ Загрузка резюме добавит данные к существующим.
Дубликаты не удаляются автоматически.
Если база уже заполнена — лучше добавлять вручную.
```

### Страница превью (`upload/preview`)
Показать что получилось ПЕРЕД сохранением:

**Блок "Личные данные":**
```
Имя: Иван Иванов
Email: ivan@example.com
...
```

**Блок "Найдено мест работы: N"**
Для каждой компании:
```
▼ Яндекс — Senior Android Developer (2021 — 2024)
    Проекты: 3
    └─ Яндекс.Карты — Android
    └─ Эксперимент с новым UI
    └─ Онбординг новичков
```

**Кнопки:**
- `✓ Сохранить в базу` → POST /upload/confirm
- `✗ Отменить` → redirect /jobs

## Обработка ошибок
- Файл не PDF/DOCX → flash ошибка
- Claude API не ответил → flash ошибка + показать сырой текст
- JSON не распарсился → логировать, показать сырой текст с предложением заполнить вручную
